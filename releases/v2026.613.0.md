# Slaw v2026.613.0

Security: instances can now present a pre-shared enrollment secret when
connecting to a Botfather control tower (audit finding C2).

## Highlights

- **Enrollment secret support.** When a tower requires a pre-shared enrollment
  secret, set `SLAW_BOTFATHER_ENROLLMENT_SECRET` (or `botfather.enrollmentSecret`
  in config) and the reporter sends it on `/enroll`. Towers that don't require a
  secret are unaffected — the field is omitted and behaviour is unchanged.

## Notes

- Prefer the env var over the config file so the secret isn't committed.
- Pairs with the tower-side `BOTFATHER_ENROLLMENT_SECRET` introduced in
  Botfather's Phase 2 enrollment-trust hardening. See the
  [self-hosting guide](https://docs.slaw.run/botfather/deploy/self-hosting).
