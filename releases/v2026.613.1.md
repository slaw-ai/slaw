# Slaw v2026.613.1

Security: webhook signature verification and cloud-tenant source-IP allowlist
(audit findings H2, H5).

## Highlights

- **Signed webhook deliveries (jira-sync).** The bundled jira-sync plugin can
  now verify inbound Jira webhooks. Set a `webhookSecretRef` on the plugin
  instance (the name of a Slaw secret holding the Jira webhook secret) and every
  delivery must carry a valid `X-Hub-Signature` (HMAC-SHA256 over the raw body)
  or it is rejected before any work is done. Instances without a secret are
  unaffected.
- **Cloud-tenant source-IP allowlist.** The trusted `x-slaw-cloud-*` header path
  (which grants instance-admin from a shared token) can now be restricted to a
  set of edge-proxy IPs/CIDRs via `SLAW_CLOUD_TENANT_ALLOWED_IPS`. Requests from
  any other source are refused, so a leaked token can't be replayed from
  elsewhere. When the token is set without an allowlist, Slaw warns at startup.

## Notes

- Configure the Jira webhook secret in Jira's webhook settings, store it as a
  Slaw secret, and point `webhookSecretRef` at it.
- Ensure your edge proxy strips inbound `x-slaw-cloud-*` headers from client
  requests — the allowlist is defence in depth, not a substitute for that.
- See the [webhook security](https://docs.slaw.run/slaw/build/plugins/jira-sync)
  docs.
